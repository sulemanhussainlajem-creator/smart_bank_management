# Security Policy

## Supported Versions

This project is currently under active development. Only the latest version in the `main` branch receives security updates.

| Version | Supported |
|---------|-----------|
| Latest (main) | ✅ |
| Older versions | ❌ |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

### How to Report

- Open a **private issue** if available, or contact the project owner directly.
- Include:
  - A description of the vulnerability.
  - Steps to reproduce the issue.
  - The affected files or modules.
  - Screenshots or proof of concept (if applicable).

### Response Time

- We aim to acknowledge reports within **3–7 days**.
- Valid security issues will be investigated and fixed as soon as possible.

### Scope

Examples of security issues include:
- SQL Injection
- Authentication or authorization bypass
- Password handling vulnerabilities
- Sensitive data exposure
- Session management flaws
- Input validation issues

We appreciate responsible disclosure and thank everyone who helps improve the security of this project.
